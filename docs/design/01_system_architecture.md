## 0. 목적

8개 모듈 책임 경계를 명확히 정의하고, 모호한 영역을 결정한 1차 설계 문서. 이 문서는 시스템 아키텍처 5단계(A→B→C→D→E)의 첫 단계 결과.

## 1. 모듈 책임 표

| # | 모듈 | 직접 하는 일 (책임) | 절대 안 하는 일 (비책임) |
|---|---|---|---|
| 1 | `backend/` | 주문 API, 견적 워크플로, DB CRUD, 계약서 자동 생성, 결제 PG(Toss) 연동, 일정 관리, 인증·권한, 알림 트리거, 운영 로그 수집 | 안무 디자인, 시뮬레이션 렌더링, 드론 직접 제어, CV 처리, 펌웨어 빌드 |
| 2 | `frontend/` | 캘린더 UI, 폼 UI, 결제 UI, 미리보기 표시, 계약서 다운로드 | 견적 산출, 안무 디자인, 비즈니스 로직, DB 직접 액세스 |
| 3 | `design-system/` | 패턴 라이브러리, 알고리즘 보간, 충돌·물리 검증, 안무 직렬화, Skybrush 연동, 3D 시뮬레이션 출력(서버사이드 영상) | DB 영구 저장, 사용자 관리, 결제, 현장 통신 |
| 4 | `logistics/` | 장비 인벤토리, 이송 일정, 배송 업체 API 연동, 현장 도착 확인 | 안무 디자인, 드론 제어, 결제 |
| 5 | `field-controller/` | 안무 다운로드, 시작·sync·비상정지 송출, 텔레메트리 수신·1차 캐시·backend로 스트리밍, 운영 대시보드, E-Stop 처리, 위치 미세 보정 명령 | 안무 디자인, 자율 안전 동작(펌웨어 책임), 결제, DB 영구 저장 |
| 6 | `safety-monitor/` | 카메라 영상 수신, 드론 detection(CV), 영역 이탈 탐지, 추락 패턴 인식, ring buffer로 영상 평시 보관, 사고 시 영상 backend 전송, field-controller에 이벤트 전달 | 드론 명령 직접 송출, 안무 디자인, DB 영구 저장 |
| 7 | `drone-firmware-l15/` | 사전 다운로드된 안무 자체 실행, LED 제어, 자율 안전 동작(저배터리·통신두절·IMU 이상), Lighthouse 위치 인식, 위치·상태 보고 | 안무 디자인, 드론 간 직접 통신, 외부 명령에 의한 자율 안전 비활성화 |
| 8 | `drone-firmware-l3/` | (L1.5 데뷔 후) 위 동일 + 자체 BOM 하드웨어 드라이버(모터·LED·라디오·IMU·광검출기) | 위 동일 |

## 2. 모호한 경계 — 결정 (락)

| # | 영역 | 결정 | 이유 |
|---|---|---|---|
| α | 안무 디자인 결과물 영구 저장 | **backend DB** | 고객·일정·계약과 연계 필요 |
| β | 3D 시뮬레이션 미리보기 렌더 | **서버사이드 영상 파일 생성** (design-system) | 클라이언트 부담 적음, 캐싱 가능 |
| γ | 장비 이송 관리 모듈화 | **logistics 독립 모듈** | 도메인 분리, 향후 외부 운송업체 통합 확장성 |
| δ | 사용자 인증·권한 | **backend 자체** | 외부 의존성 최소 |
| ε | 결제 처리 | **backend + Toss** | 한국 시장 표준 |
| ζ | 알림 (이메일·SMS·카톡) | **backend + 외부 서비스** (SendGrid·Twilio·카카오 알림톡) | 외부 서비스가 표준, 직접 구현 부담 |
| η | 드론 텔레메트리 영구 저장 | **field-controller 1차 캐시 + backend 영구 저장** | 현장 즉시 응답 + 사후 감사·분석 |
| θ | 카메라 영상 영구 저장 | **사고 시점 ±2분, 얼굴 blur 후 backend 저장, 1년 보유 후 자동 파기**. 평시는 safety-monitor ring buffer | 한국 개인정보보호법 + 사고 증거 둘 다 충족 |
| ι | 운영 로그·이벤트 | **backend DB(audit) + Sentry(에러 추적)** | 비즈니스 로그 vs 에러 추적 분리 |
| κ | `drone-firmware-l3/` 현재 처리 | **빈 폴더 + 짧은 README** ("L1.5 데뷔 후 시작" 명시) | 후순위 명시, 폴더 구조만 유지 |

## 3. 모듈 분류

### 중앙 (SaaS, 원격)
- `backend/` (1)
- `frontend/` (2)
- `design-system/` (3) ← 디자이너가 사용하지만 산출물·실행은 중앙 인프라
- `logistics/` (4)

### 현장 (이동·일회성 셋업)
- `field-controller/` (5)
- `safety-monitor/` (6)

### 디바이스 (드론 임베디드)
- `drone-firmware-l15/` (7)
- `drone-firmware-l3/` (8, 후순위)
