## 2. S1 — 드론 1대 통신 두절

**트리거**: Crazyradio link timeout (3-5초간 ACK 없음)

**감지**:
- `drone-firmware`: 펌웨어 자체 last-seen 시각 추적
- `field-controller`: 텔레메트리 끊김 감지 (I12 미수신)

**즉시 대응**:
1. **드론 펌웨어**: 5초 내 재연결 실패 → 자율 호버 → 안전 착륙 (외부 명령 무시, 자체 안전 우선)
2. `field-controller`: 운영 대시보드 알림, `safety-monitor`에 단일 드론 이벤트 통보 (gRPC)

**복구**:
- 시연 중 = 해당 드론 제외하고 진행 (29대로 안무 일부 빠짐 감수)
- 시연 후 회수, 라디오 재페어링

**예방**:
- 셋업 데이 RF 스캔 → 충돌 채널 회피 (D §2)
- 동글당 드론 8-10대 그룹화 (부하 분산, D §2a)

## 3. S2 — 다수 드론 통신 두절 (라디오 간섭)

**트리거**: 같은 동글에 묶인 드론 다수가 동시 timeout

**감지**: `field-controller`가 5초 내 동글당 50%+ 끊김 패턴 인식

**즉시 대응**:
1. `field-controller`: SW 비상정지 트리거 → 전 드론 자율 호버·착륙 시도
2. **드론 펌웨어**: 통신 두절 시 자체 호버 (S1과 동일 메커니즘)
3. 운영자: E-Stop 물리 버튼 (백업)
4. 외부 RF 간섭원 확인 (다른 부스 새 무선기기, LED 조명 토글)

**복구**:
- 채널 변경 후 재페어링 (시연 중단, 재시작 보통 어려움)
- 시연 종료 후 RF 재스캔, 채널 풀 갱신

**예방**:
- 셋업 데이 RF 스캔 + 시연 직전 재확인
- 박람회 운영진과 시연 시간대 부스 무선 사용 자제 협의

---

## 4. S3 — 드론 추락 (모터·배터리·프롭 실패)

**트리거**: 단일 부품 실패 (모터 정지, 배터리 컷오프, 프롭 파손)

**감지**:
- `drone-firmware`: IMU 이상 신호 (가속도 급변), 모터 응답 이상
- `safety-monitor`: 카메라 영상 추락 패턴 (수직 강하 검출)

**즉시 대응**:
1. **드론 펌웨어**: 모터 즉시 정지 (낙하 시 부상 최소화 — soft-fail)
2. `field-controller`: SW 비상정지 → 다른 드론 전체 안전 호버
3. 운영자: E-Stop 물리 버튼 즉시
4. `safety-monitor`: 사고 시점 ±2분 영상 `backend` 전송 (얼굴 blur 후, ADR-0001)

**복구**:
- 시연 즉시 중단, 관객·운영자 안전 확보
- 사고 보고서 작성 (audit_log), 보험·법적 대응 절차 시작
- 추락 드론 회수, 원인 분석 (블랙박스 텔레메트리 활용)

**예방**:
- 시연 전 모든 드론 사전 점검 (배터리·모터·프롭 100% 검증)
- 객석 위 비행 절대 금지 (geofence 강제)
- 부스 환경에서 Tether 옵션 적용 (천장 케블라 줄 추락 방지)

> **이 시나리오 1회 = 사업 종결 가능성**. 보험·법적 책임 구조 사전 준비 필수.

## 5. S4 — Lighthouse IR 간섭 → 위치 인식 실패

**트리거**: 박람회장 LED·레이저·강한 조명이 Lighthouse IR 신호 방해

**감지**:
- `drone-firmware`: 위치 추정 분산(covariance) 증가, 위치 불연속
- `field-controller`: 다수 드론에서 위치 이상 텔레메트리

**즉시 대응**:
1. **드론 펌웨어**: 위치 신뢰도 임계 이하 시 자율 호버 → 착륙
2. `field-controller`: SW 비상정지 (전 드론 안전 호버)

**복구**:
- 간섭원 차단 (조명 토글 OFF, 가림판 설치)
- Lighthouse 베이스 재배치 또는 추가 (사거리 보강)

**예방**:
- **셋업 데이 IR 스펙트럼 분석 — 최우선 게이트** (Phase 0 검증 필수 항목)
- Lighthouse 베이스 위치 다양화 (한 방향 차폐 회피)

> 박람회 환경에서 가장 자주 발생 가능한 안전 시나리오. Phase 0에서 IR 간섭 검증 안 끝나면 데뷔 게이트 통과 X.

## 6. S5 — field-controller PC 다운

**트리거**: 시스템 크래시 / 전원 차단 / OS 정지

**감지**:
- 드론 펌웨어: 통신 두절 (S1·S2와 동일 메커니즘)
- 운영자: 화면 freeze 직접 확인

**즉시 대응**:
1. **드론 펌웨어**: 통신 두절 N초 후 자율 호버 → 착륙 (펌웨어 자율 안전이 PC 다운에도 작동)
2. 운영자: E-Stop 물리 버튼 (PC 우회 직결 회로)
3. 백업 PC 즉시 준비

**복구**:
- 백업 PC 부팅 → 안무 다시 다운로드 → 라디오 재연결 (시연 재시작 보통 어려움, 종료 권장)

**예방**:
- **PC 이중화** (메인 + 백업)
- **UPS 무정전 전원** 필수
- 시연 직전 PC 재부팅 + 안정성 점검 (체크리스트 의무)

## 7. S6 — geofence 위반 탐지 → 비상정지

**트리거**: 드론이 정의된 안전 영역 밖으로 이동

**감지**: `safety-monitor` (CV 기반 위치 추적 + geofence 영역 비교)

**즉시 대응**:
1. `safety-monitor` → `field-controller`: 위반 이벤트 (I9, gRPC)
2. `field-controller` → 위반 드론: 즉시 안전 호버 명령 (I11, Crazyradio)
3. 다수 위반 시: 전 드론 비상정지

**복구**:
- 위반 드론 회수
- 안무 검증 재실행 (`design-system` 디자인 단계 버그 가능성)
- 시연 일부 중단 또는 종료 결정

**예방**:
- **디자인 검증 파이프라인에서 geofence 자동 체크** (안무 출력 단계)
- 시연 전 시뮬레이션으로 안전 영역 사전 확인 (Skybrush Live)

---

## 8. S7 — 객석·관객 안전 영역 침입

**트리거**: 사람이 드론 비행 영역으로 진입

**감지**:
- `safety-monitor`: CV로 사람 detection + 비행 영역 진입 판단
- 운영자: 직접 눈으로 확인 (가장 빠른 채널)

**즉시 대응**:
1. **운영자: E-Stop 물리 버튼 즉시** (가장 빠름)
2. `safety-monitor`: 자동 비상정지 트리거 (감지 즉시)
3. `field-controller`: 전 드론 안전 호버 → 착륙

**복구**:
- 침입자 안전 확보, 영역 재정리
- 시연 재시작 결정 (안전 점검 후, 또는 종료)

**예방**:
- **비행 영역 물리적 격리** (펜스·라인·안내 인력)
- 사전 안내 방송 ("드론 시연 중, 진입 금지")
- 운영자 인력 영역 경계 배치 (5-7명 운용팀 중 일부 전담)

## 9. S8-S10 짧은 메모

### S8. backend 다운 (시연 중)
- 안무는 펌웨어 자체 실행이라 **시연 자체엔 영향 없음**
- 텔레메트리 영구 저장 일시 중단 → `field-controller` 1차 캐시 보관 → 복구 후 동기화
- 사용자 웹·예약은 영향 → backend 복구 우선

### S9. 결제 실패
- Toss 결제 실패 시 backend 자동 재시도 + 사용자 알림
- 일정 확정은 결제 성공 후
- 실패 지속 시 견적 단계로 롤백

### S10. 장비 이송 지연·손상
- `logistics`가 backend에 즉시 보고 → 운영자 대응
- **백업 인벤토리 사전 준비** (예비 드론 5대, 베이스 1개)
- 행사 캔슬·연기 시 위약금·환불 절차 (계약서 명시)

## 10. 운영팀 사고 대응 체크리스트

시연 중 안전 시나리오 발생 시 **30초 이내** 절차:

1. ✅ **E-Stop 물리 버튼 누르기** (가장 빠름)
2. ✅ **관객 안전 확보** (영역 정리, 안내)
3. ✅ **사고 시점 영상 보존 확인** (`safety-monitor` 자동, 추가로 운영자 메모)
4. ✅ **외부 도움 요청** (필요 시 119, 박람회 운영진)
5. ✅ **시연 재시작 여부 결정** (운영팀장 권한, 안전 점검 후)
6. ✅ **사고 보고서 작성** (시연 종료 후 24시간 내, audit_log + 외부 보험사·법무 통보)
